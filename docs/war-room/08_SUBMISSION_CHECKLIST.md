# HyperFocus OS War Room: Submission Checklist

## Core Assets
- Product name locked
- One-line pitch locked
- Short description locked
- Screenshots ready
- Demo video ready or planned
- Repo link ready
- Live app link ready if applicable

## Demo Day Assets
- Known-good login or session
- Seeded data loaded
- Backup mode prepared
- Script printed or pinned
- One clean browser profile
- Charger, hotspot, and local backup available

## Story Check
- Problem is clear in one sentence
- Product is clear in one sentence
- Demo shows one real transformation
- Closing line is memorable

## Submission Copy Check
- No clinical claims
- No inflated security claims
- No features pitched that are not visible
- No vague "AI does everything" language

## Repo Check
- Docs are easy to find
- No secrets committed
- No build artifacts committed
- Folder structure is clean
- Stable entrypoint file exists

## Final Rehearsal Check
- Run the exact demo path 5 times
- Rehearse on the exact machine
- Rehearse with poor network once
- Rehearse the fallback story

## Judges And Reviewers
- They should understand the product fast
- They should feel the pain fast
- They should see a win fast
- They should remember the rescue moment

## Final Go / No-Go
- If the hero flow is strong, submit.
- If the extras are weak, hide them.
- If the demo is fragile, simplify it again.
