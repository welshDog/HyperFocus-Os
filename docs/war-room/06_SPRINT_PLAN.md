# HyperFocus OS War Room: 7-Day Sprint Plan

## Sprint Goal
Ship a demo-safe `HyperFocus OS` MVP that proves one transformation: overwhelm becomes one clear next step.

## Day 1: Lock Truth
- Finalize pitch, product call, and v1 scope
- Confirm hero flow and screen list
- Confirm repo shape and stack
- Freeze what is out of scope

## Day 2: Build The Shell
- Create app scaffold
- Create core layout and route structure
- Build Capture Screen
- Build Prioritized Plan Screen with seeded data

## Day 3: Build The Action Loop
- Build Focus Sprint Screen
- Build `I'm stuck` rescue path
- Add simple transition states
- Make the hero flow complete with mock data

## Day 4: Add System Logic
- Wire relay-style triage logic
- Add structured output states
- Add basic persistence where needed
- Make outputs stable enough for demo repetition

## Day 5: Polish The Demo
- Tighten copy
- Improve visual hierarchy
- Make the interface feel calm
- Add final recap screen
- Record first internal demo run

## Day 6: Harden The Build
- Add fallback paths
- Add seeded demo mode
- Fix rough edges and fragile states
- Run QA rehearsal on the exact demo path

## Day 7: Submit And Rehearse
- Freeze features
- Rehearse the live script multiple times
- Finalize screenshots, pitch copy, and links
- Prepare backup video and fallback data

## Daily Checkpoint Questions
- Does the main path still feel simple?
- Can a judge understand it in under 10 seconds?
- Is the build more reliable than it was yesterday?
- Did we accidentally add noise?

## Release Gates
- Main flow works cleanly
- Demo path survives refresh and retries
- Error states are readable
- Backup path exists
- Submission assets are ready

## If Time Breaks
- Keep Capture -> Plan -> Focus -> Win
- Cut all non-essential settings
- Cut analytics
- Cut integrations
- Cut anything that weakens the hero demo
